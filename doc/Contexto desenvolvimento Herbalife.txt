6. CHECKLIST DE SEO ON-PAGE (TODA PÁGINA)

META TAGS OBRIGATÓRIAS
──────────────────────
✓ Meta charset UTF-8
✓ Meta viewport (mobile-first)
✓ Title (50-60 caracteres com palavra-chave)
✓ Meta description (150-160 caracteres)
✓ Open Graph (og:title, og:description, og:image 1200x630, og:url)
✓ Canonical URL
✓ Robots meta (index, follow)

HIERARQUIA HTML
───────────────
✓ H1 único por página
✓ H2, H3, H4 em hierarquia lógica
✓ Alt descritivo em todas as imagens
✓ Links internos com âncora descritiva

RASTREAMENTO
─────────────
✓ Google Tag Manager (placeholder)
✓ Eventos de clique em CTA
✓ Consentimento de cookies antes de rastrear

11. CONFORMIDADE LGPD (MÍNIMA)

BANNER DE COOKIES
──────────────────
Obrigatório, simples, 2-3 linhas
"Usamos cookies para melhorar sua experiência. Ao continuar, você concorda com 
nossa Política de Privacidade."

PÁGINA /privacidade.html
─────────────────────────
Resumida, 300-400 palavras, linguagem clara
Seções: O que são cookies, Quais dados coletamos, Como usamos, Seus direitos (LGPD), 
Contato

CONSENTIMENTO EXPLÍCITO
────────────────────────
Antes de rastrear (GA4, GTM, pixel): usuário clica "Aceitar" no banner

15. COMANDOS ÚTEIS NO TERMINAL (UBUNTU)

# Criar nova página a partir do modelo
cp em_obras.html nova-pagina.html

# Buscar e substituir em lote
sed -i 's/SUBSTITUIR_TITULO/Meu Título/g' nova-pagina.html

# Validar HTML
tidy -q -e nova-pagina.html

# Comprimir imagens para WebP
for f in img/*.png; do cwebp "$f" -o "${f%.png}.webp"; done

# Deploy
git add . && git commit -m "nova pagina: [nome]" && git push

16. PROTOCOLO DE ENTREGA DE CÓDIGO

FORMATAÇÃO OBRIGATÓRIA
───────────────────────
Todo código (HTML, CSS, JavaScript, Markdown, JSON) deve ser apresentado em blocos 
de código estruturados em Markdown com tag de linguagem.

FLUXO DE CODE REVIEW
─────────────────────
O código será revisado minuciosamente pelo usuário (Engenheiro de Sistemas). Portanto:
• Código limpo, documentado com comentários breves

• Validado em HTML, responsividade, velocidade, acessibilidade

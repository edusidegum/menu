## 1. Visão Geral e Propósito

O Ecossistema HLF Pro2Col é um conjunto de páginas web, scripts e ativos digitais projetado para:
- Atrair e qualificar prospects por meio de conteúdo consultivo e geolocalizado
- Direcionar leads para conversão via WhatsApp, catálogo oficial e cadastro Herbalife
- Sustentar autoridade local em Novo Hamburgo/RS e região do Vale do Sinos
- Operar com performance, conformidade e rastreabilidade

### 1.1 Princípios Fundamentais

- **Atendimento humano assistido:** respostas rápidas com apoio de IA consultiva e atalhos de digitação — sem bots conversacionais
- **Identidade visível:** toda página exibe, acima da dobra, a identificação "Distribuidor Independente Herbalife: Edu Sidegum"
- **Uso padrão de componentes compartilhados:** CSS, scripts, header, footer e blocos legais são mantidos em diretório comum
- **Conformidade total:** disclaimer oficial Herbalife presente em todas as páginas
- **SEO local:** Novo Hamburgo/RS como âncora geográfica principal

## 5. JavaScript Compartilhado

### 5.1 Scripts do Ecossistema

| Script | Função | Carregamento |
|---|---|---|
| `bloco-incluir.js` | Injeção dinâmica de topbar, cookies, GA4 e footer | `defer` no `<head>` |
| `bloco-protecao.js` | Proteção de conteúdo (anti-cópia, bloqueio de contexto, interceptação Ctrl+C, anti-cache) | `defer` |
| `dinamico.js` | Funcionalidades interativas complementares | `defer` |
| `novo_dir_dinamico.js` | Navegação dinâmica entre páginas | `defer` |

**Localização:** `HLF/assets/js/` (dentro de cada projeto, não em diretório global)

### 5.2 Ordem de Injeção (via `bloco-incluir.js`)

1. Topbar (injetada no início do `<header>`)
2. Banner cookies/LGPD (injetado no final do `<body>`)
3. GA4 e pixel de controle (injetados no `<head>` condicionalmente)
4. Footer padrão (presente no HTML, não injetado)

### 5.3 Código do `bloco-incluir.js`

```javascript
/**
 * Injetor de Componentes Compartilhados
 * Garante que cookies, GA4 e topbar sejam injetados
 * sem bloquear o render da página principal.
 */
const BASE = '';  /* mesmo diretório do projeto */

function injetar(arquivo, seletorAlvo, posicao) {
    return fetch(arquivo + '?v=' + Date.now())
    .then(r => r.text())
    .then(html => {
        const alvo = document.querySelector(seletorAlvo);
        if (alvo) alvo.insertAdjacentHTML(posicao, html);
    })
    .catch(() => {});  // Degradação graciosa
}

// Banner de cookies — injeta no final do body
injetar('menu/compartilhado/bloco-cookies.html', 'body', 'beforeend');

// GA4 / GTM — injeta no head (quando configurado)
// injetar('menu/compartilhado/bloco-ga4-gtm.html', 'head', 'beforeend');
```

### 5.4 `bloco-protecao.js` — Mecanismos de Proteção

Ativado pela meta tag `<meta name="protecao-copia" content="ativado">`.

| # | Mecanismo | Descrição |
|---|---|---|
| 1 | Anti-cópia (seleção) | Aplica `user-select: none` no body via JavaScript |
| 2 | Bloqueio de menu de contexto | Previne clique direito (botão direito do mouse) |
| 3 | Interceptação de Ctrl+C | Substitui conteúdo copiado por "Cópia não autorizada." |
| 4 | Proteção de conteúdo textual | Impede seleção maliciosa de textos longos |
| 5 | Meta tags anti-cache | Adiciona `Cache-Control`, `Pragma`, `Expires` para evitar cache de conteúdo sensível |
| 6 | Controle de inserção de componentes | Componentes só são injetados se a meta tag correspondente existe no head |
| 7 | Prevenção de duplicação indevida | Estrutura modular com referência única — cada componente existe em UM local |

```javascript
// CSS: bloqueia seleção de texto
var style = document.createElement('style');
style.textContent = 'body{-webkit-user-select:none;-moz-user-select:none;-ms-user-select:none;user-select:none}';
document.head.appendChild(style);

// Intercepta Ctrl+C
document.addEventListener('keydown', function(e) {
    if (e.ctrlKey && (e.key==='c'||e.key==='C'||e.keyCode===67)) {
        var s=window.getSelection();
        if(s&&s.toString().length>0){ e.preventDefault(); return false; }
    }
});

// Substitui conteúdo copiado
document.addEventListener('copy', function(e) {
    e.preventDefault();
    e.clipboardData.setData('text/plain','Cópia não autorizada.');
});

// Previne menu de contexto
document.addEventListener('contextmenu', function(e) {
    e.preventDefault();
});
```

**Regras de Proteção:**
- Toda página DEVE ter a meta tag `protecao-copia` ativada, exceto páginas institucionais (política de privacidade, termos de uso)
- O comportamento pode ser adaptado conforme a sensibilidade do conteúdo da página
- O bloqueio de contexto NÃO deve impedir navegação acessível (teclado, leitores de tela)
- O sistema anti-cópia é uma barreira de integridade, não de segurança absoluta

### 5.5 Política de Privacidade, Cookies LGPD, GA4 e Pixel

- Inserção feita **exclusivamente via JavaScript**, por meio do `bloco-incluir.js`
- O banner de cookies/LGPD é apresentado como pop-up no final do body
- GA4 e pixel de controle são carregados de forma condicional, respeitando a aceitação de cookies
- A política de privacidade e os termos de uso são links fixos no footer, com páginas hospedadas em `doc/`
- **Local dos arquivos institucionais:** `HLF/doc/politicadeprivacidade.html` e `HLF/doc/termosdeusodistribuidorindependenteherbalife.html`

1. CORREÇÃO DE PATHS DOS SCRIPTS CORE

Instrução: Nos arquivos gerados, substitua o path dos scripts core:

❌ ERRADO: https://raw.githubusercontent.com/edusidegum/menu/main/scripts/bloco-incluir.js
✅ CORRETO: https://edusidegum.github.io/menu/scripts/bloco-incluir.js

Mesma regra para bloco-protecao.js.

Regras:

Usar SEMPRE edusidegum.github.io/menu/ — NUNCA raw.githubusercontent.com
A raw URL tem maior latência e pode mudar de comportamento
Colocar no <head> com atributo defer
Se colocado antes de </body> (sem defer), funciona igual mas o defer é ignorado


2. CSS RESET FORTE (IMPORTANT OVERRIDE)

Instrução: Adicionar no início do style.css de TODO projeto:

body {
  background-color: #0a0e17 !important;
  color: #e2e8f0;
}

.card, .card-body, [class*="card"] {
  color: #e2e8f0 !important;
}

Justificativa: O HTML gerado frequentemente insere estilos inline com #ffffff ou #f8fafc que sobrescrevem o tema dark. O !important no body e cards garante consistência visual.


3. ABORDAGEM PEDAGÓGICA — NUNCA LINK DIRETO

Instrução: Em páginas educacionais de manutenção/suporte:

❌ NUNCA oferecer download de driver, software ou script executável como ação principal
✅ ENSINAR o método manual primeiro: "Win+R → cmd → [comando]"
✅ Oferecer download .bat/script APENAS como alternativa, após ênfase na execução manual
✅ Em páginas públicas (dashboard, index): remover completamente qualquer oferta de download

Exemplo implementado (manutencao.html):

4. POST-IT INTERATIVO — ESPECIFICAÇÃO TÉCNICA

Instrução: Para implementar cards que abrem post-it ao clicar:

HTML do overlay (antes de </body>)

<div class="postit-overlay" id="postitOverlay">
  <div class="postit" id="postitContent">
    <button class="close-btn" id="closePostit">✕</button>
    <div class="cmd-text" id="postitCmd"></div>
    <div class="desc-text" id="postitDesc"></div>
  </div>
</div>

CSS do post-it

.postit-overlay {
  position: fixed; top: 0; left: 0; width: 100%; height: 100%;
  background: rgba(0,0,0,0.7);
  display: flex; align-items: center; justify-content: center;
  z-index: 9999;
  opacity: 0; visibility: hidden;
  transition: opacity 0.3s ease, visibility 0.3s ease;
}
.postit-overlay.active { opacity: 1; visibility: visible; }
.postit {
  background: #fef9c3;
  width: 380px; max-width: 90%;
  padding: 28px 24px;
  border-radius: 4px;
  box-shadow: 5px 5px 15px rgba(0,0,0,0.3);
  font-family: 'Caveat', cursive;
  font-size: 1.5rem;
  color: #0a1628;
  line-height: 1.5;
  position: relative;
  transform: rotate(var(--rot, 0deg));
}
.postit .cmd-text { font-weight: 700; font-size: 1.6rem; margin-bottom: 8px; }
.postit .desc-text { font-size: 1.3rem; white-space: pre-line; }

JavaScript (único bloco no final do body)

const commands = [
  { title: "CHKDSK", command: "chkdsk C: /f /r /x", description: "Verifica e repara erros no disco..." },
  // ... 8 comandos
];

const overlay = document.getElementById('postitOverlay');
const cmdEl = document.getElementById('postitCmd');
const descEl = document.getElementById('postitDesc');

function openPostIt(index) {
  const data = commands[index];
  cmdEl.textContent = data.command;
  descEl.textContent = data.description;
  const rot = (Math.random() * 6 - 3).toFixed(1);
  document.getElementById('postitContent').style.setProperty('--rot', rot + 'deg');
  overlay.classList.add('active');
}

function closePostIt() { overlay.classList.remove('active'); }

document.querySelectorAll('.card-item').forEach(card => {
  card.addEventListener('click', function() {
    openPostIt(parseInt(this.dataset.index));
  });
});

overlay.addEventListener('click', e => { if (e.target === overlay) closePostIt(); });
document.getElementById('closePostit').addEventListener('click', closePostIt);
document.addEventListener('keydown', e => { if (e.key === 'Escape') closePostIt(); });

Regra: Google Fonts 'Caveat' deve ser carregado no <head> como <link>, NUNCA dentro de <style>.


5. PÁGINA DE REDIRECT COM SEO (INDEX.HTML)

Instrução: Toda página de redirect (meta refresh) DEVE conter no <head>:

<title>Windows 11 Maintenance Hub — Título Descritivo</title>
<meta name="description" content="Descrição com 150-160 caracteres para SEO.">
<meta name="robots" content="index, follow, max-snippet:-1, max-image-preview:large, max-video-preview:-1">
<meta name="googlebot" content="index, follow, max-snippet:-1, max-image-preview:large, max-video-preview:-1">
<meta name="bingbot" content="index, follow, max-snippet:-1, max-image-preview:large">
<link rel="canonical" href="https://edusidegum.github.io/PROJETO/index.html">
<link rel="alternate" hreflang="pt-BR" href="https://edusidegum.github.io/PROJETO/index.html">
<meta http-equiv="refresh" content="0;url=dashboard.html">

<meta property="og:title" content="...">
<meta property="og:description" content="...">
<meta property="og:image" content="https://edusidegum.github.io/PROJETO/assets/img/og-image.webp">
<meta property="og:url" content="...">
<meta property="og:locale" content="pt_BR">

JSON-LD obrigatório: Person (Eduardo Sidegum) + WebSite + WebPage + BreadcrumbList.

Funcionamento: O Googlebot lê as meta tags do <head> ANTES de seguir o redirect. Por isso o redirect instantâneo (content="0") não prejudica a indexação — desde que as tags estejam corretas.


6. SITEMAP.XML PRÓPRIO POR PROJETO

Instrução: Cada repositório DEVE ter seu próprio sitemap.xml na raiz:

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:xhtml="http://www.w3.org/1999/xhtml">
  <url>
    <loc>https://edusidegum.github.io/PROJETO/pagina.html</loc>
    <changefreq>weekly</changefreq>
    <priority>0.9</priority>
  </url>
</urlset>

Regras:

Nome: sitemap.xml (minúsculo) — NÃO sItemap.xml
Incluir APENAS páginas com index, follow
Prioridade: principal 1.0, conteúdo 0.9, secundárias 0.7-0.8
Changefreq: weekly para hubs, monthly para conteúdo estável


7. OTIMIZAÇÃO DE PERFORMANCE

7.1 YouTube Embed — Thumbnail Lazy Load

Substituir iframe direto por thumbnail + clique para carregar:

<div class="video-wrapper" style="position:relative;cursor:pointer;"
     onclick="this.innerHTML='<iframe style=\"position:absolute;top:0;left:0;width:100%;height:100%;\" src=\"https://www.youtube.com/embed/ID?autoplay=1\" allow=\"autoplay\" allowfullscreen></iframe>'">
  <img src="https://img.youtube.com/vi/ID/hqdefault.jpg" 
       alt="Título do Vídeo" style="position:absolute;top:0;left:0;width:100%;height:100%;object-fit:cover;border-radius:12px;">
  <div style="position:absolute;top:50%;left:50%;transform:translate(-50%,-50%);font-size:4rem;">▶️</div>
</div>

Ganho: -940KB de CSS/JS não utilizado, TBT cai de ~280ms para <100ms.

7.2 Animação do Hero — Compositável

❌ ERRADO (causa reflow):

@keyframes gradientShift {
  0% { background-position: 0% 50%; }
  50% { background-position: 100% 50%; }
}

✅ CORRETO (compositável via GPU):

.hero-section {
  background: linear-gradient(135deg, #0f172a 0%, #1e293b 50%, #0f172a 100%);
  position: relative;
  overflow: hidden;
}
.hero-section::before {
  content: '';
  position: absolute; inset: 0;
  background: linear-gradient(135deg, transparent 0%, rgba(37,99,235,0.03) 50%, transparent 100%);
  background-size: 200% 100%;
  animation: gradientShift 8s ease-in-out infinite;
  pointer-events: none;
}
@keyframes gradientShift {
  0% { transform: translateX(-50%); }
  50% { transform: translateX(50%); }
  100% { transform: translateX(-50%); }
}
